给 Gridea+github 博客添加 gitalk 评论插件

2023-01-04 2 min read # 博客 # Gridea

新年快乐！

概述

Gitalk 是一个基于 GitHub Issue 和 Preact 开发的评论插件。
功能：

  • 使用 GitHub 登录
  • 支持多语言（en, zh-CN, zh-TW, es-ES, fr, ru）
  • 支持个人或组织
  • 无干扰模式（设置 distractionFreeMode 为 true 开启）
  • 快捷键提交评论 （cmd|ctrl + enter）

鉴于博客由 github 存储仓库，用 gitalk 会方便一些。

配置 github —— 创建 gitalk 开放授权应用

  1. 登录Github网站，鼠标点击头像，点击下拉菜单的【settings】。
  2. 跳转新页面往下滑，点击左下角栏目中的开发者设置【Developer settings】。
  3. 跳转新页面，点击左侧栏的开放授权应用【OAuth Apps】，点击右侧新建【New OAuth App】。
  4. 如图操作，最后点击绿色按钮注册应用。
  5. 跳转新页面，点击右侧白色按钮【Generate a new client secret】生成一个新密码。
  6. 保存Client ID和Client Secret。

配置Gridea

  1. 打开Gridea客户端，按下图操作。
  2. 保存之后进行远程链接和同步操作，打开实际网站，由于gitalk需要发布者进行初始化，在评论栏目点击进行登录即可。

遇到的问题

  1. 博客评论区出错，【未找到相关的 Issues 进行评论请联系 @_____ 初始化创建】
    解决办法：
    一，可能站点仓库Settings中未打开issue，打开即可。

    二，可能【New OAuth App】中主页链接和授权回调链接设置错误，重新设置。

    比如我把https://erhuChenChen.github.io设置成了https://github.com/erhuChenChen/erhuChenChen.github.io，导致错误。
